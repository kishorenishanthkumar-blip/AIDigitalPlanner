01 — Abstract
Summary of the Invention
A multi-agent artificial intelligence system that transforms unstructured discovery-phase input — including stakeholder interviews, meeting transcripts, pain points, and business goals — into structured, platform-ready software requirements automatically and without human authoring.
The present invention discloses an Agentic AI Requirements Studio comprising a coordinated pipeline of six specialised artificial intelligence agents: a Discovery Parser, an Epic Generator, a Story Writer, an Acceptance Criteria Agent, an Estimation Agent, and a Platform Mapper. These agents operate in orchestrated sequence to produce complete software engineering artefacts — including epics, user stories, Given/When/Then acceptance criteria, Fibonacci story-point estimates, and priority classifications — formatted for direct import into Atlassian Jira and Rally (Broadcom).
03 — Patent Claims
Claims
1
Independent — System
A computer-implemented agentic artificial intelligence system for automated software requirements generation, comprising: a discovery input module; an orchestration controller coordinating a sequential pipeline of specialised AI agents; and a platform export module emitting structured requirements data for import into at least one enterprise project management system.
2
Independent — Method
A computer-implemented method comprising: receiving unstructured discovery input; processing through a multi-agent AI pipeline; generating epics, user stories, acceptance criteria, and effort estimates; and outputting in Atlassian Jira or Broadcom Rally format.
3
Dependent — Discovery Parser
The system of claim 1, wherein the pipeline includes a Discovery Parser Agent that extracts themes, entities, pain points, business goals, and technical constraints from unstructured input using LLM inference.
4
Dependent — Epic Generator
The system of claim 1, wherein the pipeline includes an Epic Generator Agent producing epics with title, description, business value, priority {Critical, High, Medium, Low}, and component labels.
5
Dependent — Story Writer
The system of claim 1, wherein the pipeline includes a Story Writer Agent generating user stories in canonical format "As a [persona], I want [action] so that [benefit]" with domain-specific personas.
6
Dependent — Acceptance Criteria
The system of claim 1, wherein the pipeline includes an Acceptance Criteria Agent producing Given/When/Then criteria per story, contextually specific to domain, persona, and action.
7
Dependent — Estimation
The system of claim 1, wherein the pipeline includes an Estimation Agent assigning Fibonacci story points {1,2,3,5,8,13} using domain-context-aware inference on complexity and technical risk.
8
Dependent — Export
The system of claim 1, wherein the export module simultaneously emits Jira Cloud JSON (projects[].issues[]) and Rally Agile Central Feature/UserStory bulk upload JSON.
9
Dependent — Orchestration
The method of claim 2, wherein a supervisor controller manages inter-agent handoffs, enforces schema validation, and provides real-time animated progress UI.
10
Dependent — Traceability
The method of claim 2, further comprising a bi-directional requirements traceability matrix linking each story to: (a) originating discovery input; (b) parent epic; (c) acceptance criteria.
