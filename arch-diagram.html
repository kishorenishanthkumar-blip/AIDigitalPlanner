Fleet architecture — full flow diagram

Every worker in the AIDP fleet and how they wire together at runtime: the master router fans out to agents, the orchestrator drives the lifecycle and writes the lineage graph, the testing master dispatches eight disciplines to real CI tooling, knowledge is shared memory, and every escalation is chained into the audit ledger. Solid arrows are in-process service bindings; dashed arrows cross to GitHub Actions CI.

Solid arrows = Cloudflare service bindings (worker-to-worker, in-process). Dashed = cross to GitHub Actions CI (the disciplines orchestrate-and-dispatch real tools, then post results back). The orchestrator is the only writer of the lineage graph; insights reads it; exception holds the immutable ledger.
